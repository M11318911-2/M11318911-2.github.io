Hermes vs OpenClaw

資料來源：NousResearch GitHub、Petronella Cybersecurity News、Remote OpenClaw、Wikipedia

🏷️ 兩者基本定位

🟣 Hermes — Nous Research

  • 核心理念：「會成長的 AI」— 內建學習循環
  • 定位：自我改善型 AI Agent，會從經驗中建立並改善技能
  • 最大亮點：唯一內建「用經驗自我學習」機制的 Agent
  • 開源協議：MIT（完全免費開源）
  • 最低硬體需求：$5 VPS 或本地環境

🔵 OpenClaw — Marco Steinhuber

  • 核心理念：「開源自主助理」— 跨服務自動工作流
  • 定位：讓 LLM 成為 24/7 自主助理，運作在自有硬體上
  • 最大亮點：Skills 生態系（ClawHub 100+ Skills）+ 多平台接入
  • 開源協議：開源免費（2026/02 非營利基金會接管）
  • 最低硬體需求：需運行在實體機器/VPS 上

💡 核心邏輯：用 OpenClaw（貴但聰明）做需要複雜判斷的事，用 Hermes（便宜且會學習）做大量的執行工作。省下 API 成本，同時讓系統隨著使用越來越強。

四大實際應用場景：

① 故障自動修復：OpenClaw 掛了 → Hermes 瞬間接手修復

② 規劃執行一條龍：OpenClaw 規劃 → Hermes 寫程式 → 省 50%+ API 成本

③ 全天候走廊巡查：Hermes 每 2 小時自動檢查，有錯立即通知 OpenClaw

④ 共同完美企劃：Hermes 寫初稿 → OpenClaw 修改 → 自動互相檢視盲點
