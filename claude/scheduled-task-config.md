# AI Agent 每日資訊看板 - 排程任務配置

## 📋 任務概述

**任務名稱**: AI Agent 每日資訊看板
**執行時間**: 每天早上 7:30 (本地時區)
**執行頻率**: 每日自動執行
**輸出位置**: `/sessions/exciting-sweet-allen/mnt/outputs/ai-agent-dashboard.html`

---

## 🎯 任務內容詳細說明

### 三個 Subagent 的分工

#### **Agent 1: 各家推出的技術更新**
- **搜尋目標品牌**:
  - OpenAI（GPT 系列新版本、功能更新）
  - Google Gemini（模型升級、新功能發布）
  - Claude / Anthropic（Agent SDK、新模型發布）
  - OpenClaw 及其他開源框架

- **搜尋內容**:
  - 新技術或功能的發表消息
  - 現有技術的功能更新公告
  - 官方 Blog 和新聞稿
  - 主要科技媒體的評測報導

#### **Agent 2: 各媒體的試用心得**
- **搜尋對象**:
  - 科技媒體（TechCrunch、The Verge、36 氪等）
  - YouTuber 和技術博主的體驗分享
  - Medium、知乎等專業內容平台
  - 開發者社群（Dev.to、GitHub Discussions 等）

- **搜尋重點**:
  - 新功能的深度試用報告
  - 官方或個人推薦的實用技巧
  - **優先選擇初學者友善的教學內容**
  - 對比測試與性能評估

#### **Agent 3: 新產品的介紹**
- **關注新品**:
  - Hermes Agent（自進化 AI 助理）
  - MaxHermes（雲端版本）
  - 其他新興 Agent 框架和工具
  - 開源專案的重要版本發布（v0.7.0、v1.0.0 等）

- **搜尋內容**:
  - 官方產品介紹和功能說明
  - GitHub Stars 趨勢和社群反應
  - 安裝配置和快速上手指南
  - **優先收錄初學者導向的教學**

## 📊 Dashboard 輸出規格

### 版面設計
- ✨ 現代明亮且專業簡潔的風格
- 🎨 使用 Tailwind CSS 實現響應式設計
- 💎 採用毛玻璃效果和漸層背景
- 📱 完整支援桌面和行動設備

### 資訊卡片結構
每則資訊卡片包含：
```
[標題（可點擊連結到原始來源）]
[來源網站名稱]  [發布日期]
[一句話摘要]
```

### 排序與篩選規則
- 每個區塊最多顯示 **5 則資訊**
- 依**發布日期由新到舊**排列
- 如某區塊無新消息，顯示「本週暫無新消息」
- 搜尋範圍：**過去 7 天內發布**

### 頁面元素
- 頂部：「AI Agent 每日資訊看板」標題 + 更新時間
- 三大區塊：對應三個 Agent 的搜尋結果
- 底部：最後更新時間戳記

## 📋 執行流程

1. **並行搜尋階段**
   - 3 個 Subagent 同時執行搜尋任務
   - 每個 Agent 獨立調用 WebSearch 工具
   - 預期運行時間：3-5 分鐘

2. **資料整理階段**
   - 去重與驗證資訊準確性
   - 提取必要資訊（標題、URL、來源、日期、摘要）
   - 按發布日期排序

3. **Dashboard 生成階段**
   - 將整理後的資料代入 HTML 模板
   - 生成互動式頁面
   - 保存到 outputs 目錄

4. **驗證與發布**
   - 檢查頁面渲染效果
   - 驗證所有連結有效性
   - 更新時間戳記

## 📝 注意事項

1. **資訊準確性**: 優先選擇官方來源和知名科技媒體的報導
2. **去重機制**: 同一則新聞若在多個來源出現，僅保留最早的原始報導連結
3. **初學者友善**: 收錄教學內容時，優先選擇詳細、循序漸進的入門指南
4. **更新頻率**: 如某天無新消息，保留上次的結果或顯示「本週暫無新消息」
5. **連結有效性**: 定期檢查外部連結是否仍有效
